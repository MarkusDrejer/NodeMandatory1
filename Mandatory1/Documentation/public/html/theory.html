Variable Declaration

var

A variable type which is globally scoped when outside of a function and function scoped when inside a function.

var works with hoisting and can therfore be declared after usage.

A var can be redeclared which can result in unpredicted behavior if a redeclaration takes place in a different scope.

let

let comes as an improvment over the var declaration and will solve the scoping problems.

is block scoped and only lives within a set of curly braces {}, let is allowed to be updated but not redeclared.

The same variable name can therefore only be used several times if they are within different scopes, it will otherwise result in a 'SyntaxError.'

Unlike var, let will not work properly with hoising, this is because where var is initialized as 'undefined', let has no initialization, so trying to use it with hoising will result in a 'ReferenceError'.

const

A variable declaration with const will maintain a constant value, this means a const cannot be updated nor redeclared.

Like with let, const is also block scoped and can only be accessed within the block it was declared.

Because we are unable to update a const, it must always be initialized by the time of declaration.

Even though it is impossible to update the direct value of a const, the properties of the objects can be updated.

Hoisting of const work in the same way as let, as a const is also not initialized.

Strings

In Javascript we have several valid ways of writing Strings, these include:
Double quotes = " "
Single quotes = ' '
Backticks = ` `

The reasoning for all these different ways is primarily so that a text which might include quoting can still be done in a proper fashion and if you know you need to use double quotes as part of the text you can pick one of the other options for your String, as to not interfere with the declaration of the String itself.

Backticks also has a special feature as it supports direct insertion of variables into the String by writing ${<variable>}, this is not supported by double or single quotes.

String concatenation works in two ways in Javascript, either by applying the '+' operator or by using ',' between the two.
Using '+' can lead to problems as it will completly disregard types and turn the entire thing into a String, whereas using ',' will keep the types, usually making it the better choice.

JSON

JSON stands for Javascript Object Notation and is used as a lightweight format for storing and transporting data.
It is often used to send data from servers to webpages.

The data kept in a JSON object is held in 'Key-Value pairs', meaning that every value will have a key which points to it, and to get that value you have to get it via its key, all the pairs are seperated by commas.

To annotate a JSON object you must start with curly braces: {}, otherwise it will not be valid JSON.
Both the key and value must be wrapped as a String as this is the only format JSON will support

JSON is the main component in REST as it will focus only on returning data to the user nothing else, this makes JSON optimal since its easily readable by both humans and machines and can contain vast amounts of data.

Requests

As part of the build of a REST API we have a set of typically/standarly used HTTP methods, that cover the bases of the data we want to be able to get and manipulate.

GET: the GET request is the one which a user has direct access to as it happens via the URL-field in their browser, this request is used to get data out of the API and return it to the user in a JSON format.

POST: this is used to create new instances of data, like a new user by means of the request body, it is also usually used on login.

PUT: is used when updating an already existing item in the data, like updating the price of a product on a sales site.

DELETE: will be used when deleting an existing item.

All of these different request types can be used on a collection basis or singular basis, for instance you can GET all users or use an id to only GET one specfic user.

Loops

In Javascript there a couple of different loop types to work with, as with many languages While and For are the two most common ones.

The While loop is pretty uninteresting and works the same way here as in Java, with the While keyword followed by the condition.

For loops are a bit more interesting, here we deal with a more standard style as found in Java with a variable, condition and increment.
But we can also make use of so called 'Enhanced' For loops which in a shorter manner can get either index or elements out of an array

for x in array: is an enhanced version of picking out the elements of the array and storing them in the variable x.

for x of array: will instead run through the index of the array and store it in x.

Thirdly it is possible to use a special foreach loop by writing: array.foreach(function), which will run through every element of the array and run the given function on each element.

Functions

Functions are one of the fundamental building blocks in JavaScript.
A function is a JavaScript procedure, a set of statements that performs a task or calculates a value.
To use a function, you must define it somewhere in the scope from which you wish to call it.

Standard Function

The standard function in Javascript consists of the function keyword followed by the name, parameters and the actual code to be executed.

However this way of defining functions is a bit old and comes with some problems.
The primary problem here is when using the 'this' keyword which is not closed to the function and can instead refer to higher scoped items, not just items present in the function itself, meaning you might get a wrong reference when using 'this'.

Arrow Function

A newer way of writing functions is by using what is called Arrow functions, this is basically a lambda expression of a function as it has no name and is as such anonomys.

Arrow functions start with the paranthesis with parameters followed by the '=>' arrow part and then the code to be executed.

Arrow functions are primarily easier to write and also fix the previous 'this' problem with the standard functions, because 'this' keyword is only valid within the function itself keeping it from creating problems.

Collections

When talking about collections in Javascript we really only talk about one type, Arrays.
This is because arrays in Javascript fulfills pretty much all the areas we need to cover in most cases.

Arrays in Javascript works much like in other languages, but has some extra interesting features:
It supports push and pop like with a Stack from Java, with pop also being able to take a specfic index.
It supports the spread(...) operator which will take all the values from the array and 'spreads' them into for instance another array for easy copying of elements.
The arrays can also hold JSON objects.

And much more advance functions can be applied to arrays like search, filter etc. all of which makes arrays very versatile in Javascript.

Importing

In order to use various libraries as part of our node projects, it is imperative that we import our libraries just like with most other languages.

As part of node we do not use the more common keyword 'import' which does seem like the most natural way to perform an import, instead we use the 'require' keyword.

require is actually a function which takes the library to be imported as a parameter.
require is also available on a global scope and does not need to be required by itself.

Dependencies

All dependencies of a node project is handled either through the local package.json file or by globally installed packages.
This means that all newly created project will have to re-install dependencies and packages (like Express) if they aren't already available on a global scope.

package.json

As said the local dependencies are managed via the projects package.json file, however this file does not come with newly made node projects, instead it is important that you yourself create this file and make it a valid JSON file before starting to manage and trying to install dependencies.

This file is therefore one of the most important things to remember to add, or get by running the npm init command which will create the file after setup.
