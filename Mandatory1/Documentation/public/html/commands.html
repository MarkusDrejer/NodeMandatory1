Documentation: Commands

Theory Tools Commands

cd

Stands for 'change directory' and as the name describes it is used to change the current directory.

cd <path>

To navigate through your directories use cd with the path of your destination.
The path is not case sensitive.

cd ..

When you have gone in too far or need to move back up a directory use cd ..

cd/

Only using / with cd will move you all the way up to the root directory.

node

The node command will try to load module located in the folder you pass, and start whatever is written in "main" section of package.json.

node <path>/<filename>

In order to run a node application you prefix with node followed by the path to the file you wish to run, lastly write the filename plus its extension.

node <filename>

If your terminal is already on the desired path of the file to be run, the path can be omitted from the command and the filename can be directly used.

npm

Using the npm command to install packages is very easy, in order to install packages into a project a package.json file to handle the dependencies must be present.

npm install <package_name>

To install a package you need the exact package name, after it is as simple as writing npm install followed by that package name, this will install the package locally in your current standing directory.

npm install

If one or more dependencies have been manually written in package.json simply using npm install will install all dependencies defined within and it will make sure that it is the newest version.

npm install <package_name> -g

Like with previous commands this will also install a package, however where the other installed the packages locally, by using -g, this will install the package globally directly on your system for all your other projects to use.
The -g can also be placed before the package name.

npm i

npm i, is simply used as a shortend version of npm install and will work in all cases where you would normally use npm install.

npm run <script_name>

If you have any declared scripts in your package.json file these can be run be using npm run with the name of the script, scripts are can be used for quick access to commands and to make use of enviroment variables.

npm init

It is possible to initialize npm directly in your project by using npm init this will take you through a guide of several items to fill in, like version, name etc.
When the setup is complete, a package.json will be created with these settings filled into it, making manually creating and filling not needed.

npm uninstall <package_name> (-g)

The exact opposite of install, this command will uninstall any package that you have previously installed and remove it from your dependencies.
It is also possible to add -g to uninstall a globally installed package.

ls

The ls command will show all the files, folders etc which exist on a given path, this means that you are capable of getting an overview over all items on a path after using cd, or just in general.

dir

When in a Windows command prompt, ls is not a recoqnized as a command, this is because ls is a UNIX based command, here the command dir will accomplish the same task.

The exception here is that Windows Powershell will recoqnize ls as a command.

nodemon

nodemon will make sure that your app is restarted everytime a change is detected, usefull for development, requires install via npm or to be run via npx.

nodemon <path>/<filename>

To run using nodemon use the path to the file you wish to execute and finish with the name of the file plus its extension.

nodemon <filename>

As with the node command, if the terminal is already correctly placed, the path can be omitted from the command.

nodemon.cmd <filename>

If using Windows Powershell the nodemon command differs slightly and needs the added .cmd to be run, the previous commands are based on UNIX.

budo

budo is used for rapid web development and can enable live reload and is very usefull on work-in-progress sites.

budo <path>/<filename>

The budo command also needs to be directed through your directories to where your file to be run is located, followed by the filename and its extension.

budo <filename>

As before the path is unnecessary if you are located where you have direct access to the files you wish to run.

flags

budo is often used with various extra features enabled, these are used via what is know as flags.
A flag is added after the filename and needs to be spaced, The flags can be used simultaneously with space between them, here we will list the most common used ones.

--live

For live refresh and updates as work is being performed, budo will actually not run live-reload without this flag.

--open

Will automatically open your browser on the correct address when the project spins up, this address is your machines local ip and port 9966.

npx

npx can be used to make a one-time execution of either a node_modules package or from a central cache, intalling packages needed for it to run.

npx <package_name>

In order to use npx, add npx infront of the execution of a different package that you wish to use, after which you simply follow the execution of said package.
